# 验收检查清单

## 一、网页审阅

- 是否已经完成三轮审阅
- 是否已经确认所有页面、导航、标签页、弹窗入口
- 是否已经记录关键按钮、筛选器、图表、表格、状态标签、表单字段和反馈信息
- 是否已经验证主要交互，而不是只看静态截图

## 二、事实整理

- 是否已经先生成事实型中间稿
- 中间稿内容是否只来自目标网页或目标项目
- 是否没有写入未核查到的后台能力、协议、数据库或算法内容

## 三、正式说明书内容

- 是否已经采用正式说明书结构，而不是简单功能清单
- 是否包含引言、系统综述、系统使用前提、功能模块说明、典型业务使用路径和说明边界
- 是否对每个主要模块写清定位、构成、操作、业务价值和上下游关系
- 如果用户提供了范文，当前稿件是否已经对齐其章节层次和段落密度
- 如果用户没有提供范文，当前稿件是否达到内置送审版的最低厚度

## 四、厚度检查

- 每个主要模块是否都有独立成段的完整说明，而不是一两句带过
- 概览页是否写清卡片区、图表区、趋势区、告警区和观察价值
- 列表页是否写清筛选区、字段、行级操作和业务作用
- 表单、弹窗或抽屉是否写清入口、字段、反馈和返回路径
- 典型业务路径是否不是一句话概括，而是写出先后顺序和使用目的

## 五、中文与编码

- 是否严格使用简体中文
- 是否没有乱码、替换字符、异常问号或日文假名
- 是否所有 Markdown 和脚本文件都按 UTF-8 保存

## 六、Word 成品

- 是否已经备份旧版
- 是否有封面
- 是否有目录
- 是否套用正式标题样式
- 是否没有把 `#`、`##`、`###`、`**` 或 Markdown 表格原样带进正文
- 是否排版紧凑、连续、无异常大块空白

## 七、渲染检查

- 是否已经导出 PDF
- 是否抽查了封面、目录、前段、中段、后段和末页
- 标题是否正常
- 中文是否正常
- 表格和正文是否连续，没有明显截断、错页或格式错乱

## 八、最终终审

- 是否已经重新阅读全文至少一遍
- 是否已经重新检查最终 DOCX 或 PDF 至少一遍
- 是否已经逐条核对用户的最新要求
- 是否确认内容准确、格式准确，并且与用户要求完全贴合
