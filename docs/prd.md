## 1. 应用概述

### 1.1 应用名称
Luna AI 多平台素材包生成工作台

### 1.2 应用描述
Luna 是一款面向自媒体创作者的 AI 内容生成微信小程序，提供多平台素材包生成、AI 对话、会员充值、素材管理等核心功能，帮助用户实现多平台内容生产自动化。

## 2. 用户与使用场景

### 2.1 目标用户
自媒体创作者、内容运营人员、小红书/抖音/视频号/公众号等平台创作者

### 2.2 核心使用场景
- 基于已有素材生成多平台内容素材包
- 基于行业方向和热点生成多平台内容素材包
- 通过 AI 对话获取内容创作建议
- 管理创作余额与充值记录
- 管理上传素材与生成结果
- 获取平台功能介绍与在线客服支持
- 单独充值算力进行内容生成

## 4. 业务规则与逻辑

### 4.1 用户权限与数据隔离
- 每个用户通过微信 OpenID 与后端 User ID 映射
- 不同用户间数据完全隔离
- 用户仅可查看和操作自己的数据

### 4.2 素材包生成流程规则
- 用户提交素材包生成请求后，调用保险层 Edge Function luna_guardian
- luna_guardian 判断用户意图与请求类型
- luna_guardian 执行安保检查，根据产品边界规则判断是否允许、拦截或转向
- luna_guardian 整理用户输入，补全缺失字段
- luna_guardian 根据请求类型选择 Prompt 模板
- luna_guardian 调用现有 arkclaw_chat Edge Function（内部 fetch 调用）
- luna_guardian 检查返回结果是否完整，结果格式不完整时做轻量修复或返回友好提示
- luna_guardian 将结果保存到 materials 表（type=work）
- 前端跳转至素材包结果页展示生成结果

### 4.3 Luna 产品边界定义
- Luna 是一个会自然对话的多平台内容创作助手
- 负责理解用户创作需求、组织素材、生成内容方案和投放建议
- 不负责登录平台账号、采集后台数据、绕过风控、自动发布或自动运营账号

### 4.4 Luna 必须允许的功能
1. 自然语言聊天
2. 小红书文案
3. 抖音脚本
4. 视频号口播
5. 公众号文章
6. 多平台素材包
7. 方向热点分析
8. 内容矩阵
9. 账号定位
10. 人设规划
11. 封面建议
12. 话题标签
13. 发布时间建议
14. 投放建议
15. 基于用户上传截图、表格、文案、图片、公开链接做分析

### 4.5 Luna 必须禁止的功能
1. 登录小红书/抖音账号
2. 扫码登录
3. Cookie / 登录态搬运
4. 代理 / 动态代理 / 绕风控
5. 采集后台数据
6. 获取私有播放量、访客、曝光、转化数据
7. 自动发布
8. 自动推送
9. 自动评论
10. 自动私信
11. 批量养号
12. 刷播放、刷粉、刷赞
13. 声称已经读取平台后台数据

### 4.6 安保层触发规律
- 平台词 + 创作词 = 放行（allow）
- 平台词 + 账号/采集/自动化词 = 拦截（block）
- 平台词 + 数据分析词 + 用户提供数据 = 放行（allow）
- 平台词 + 数据分析词 + 要 Luna 去抓后台 = 拦截（block）

### 4.7 安保层白名单
- 平台白名单：小红书、抖音、视频号、公众号、快手、微博、B站、知乎
- 创作动作白名单：写、生成、改写、润色、优化、起标题、想选题、做脚本、做口播、做封面建议、做标签、做投放建议、做内容规划、做素材包、做内容矩阵、做账号定位
- 内容对象白名单：文案、标题、脚本、选题、口播稿、封面、图片提示词、素材包、内容方案、推广方案、投放建议、标签、话题、栏目、账号定位、人设、卖点、产品介绍、活动信息、品牌资料、用户画像、竞品截图、公开链接、手动上传的数据
- 数据来源白名单：用户手动输入、用户上传图片/视频/文案/表格/截图、用户提供公开链接、公开网页信息、搜索引擎公开结果、平台公开可访问页面、公开榜单/新闻/文章、历史生成结果、素材库里的用户素材

### 4.8 安保层处理动作
1. allow_chat：普通放行，正常对话回复
2. allow_generate：允许生成素材包/内容方案
3. ask_clarify：追问澄清，当意图不明确时
4. safe_redirect：安全转向，当触碰禁止范围时，统一话术：「当前 Luna 不支持账号登录、Cookie、代理、后台数据采集、自动发布或自动推送。你可以上传素材，或输入内容方向，我可以基于你提供的素材、公开信息和平台内容规律，帮你生成多平台素材包和投放建议。」
5. sanitize_output：输出修正，当生成内容中出现敏感描述时静默修正

### 4.9 安保层架构规则
- 明确命中白名单 → 直接放行（allow_chat / allow_generate），不调 LLM，提升速度
- 明确命中黑名单 regex → 安全转向（safe_redirect），不调 LLM
- 灰色地带（无法直接判断）→ 调 LLM 进行语义判断，由 LLM 返回处理动作
- LLM 判断时要以白名单保护正常创作体验为优先，只有确定越界才拦截

### 4.10 安保层核心原则
- Luna 主入口必须是自然语言对话，不能被改成死流程表单
- 安保层不能过度拦截正常创作需求
- 白名单优先，黑名单只拦截账号控制/后台采集/绕风控/自动化发布

### 4.11 套餐配额监控与熔断规则
- 系统实时监控用户实际消耗的素材包生成次数、趋势研究次数、图片生成数量、视频生成时长
- 每次生成请求完成后，记录实际消耗量并累加至用户账户
- 当用户素材包生成次数超过套餐配额时，触发素材包生成功能熔断，禁止继续生成
- 当用户趋势研究次数超过套餐配额时，触发趋势研究功能熔断，禁止继续使用
- 当用户图片生成数量超过套餐配额时，触发图片生成功能熔断，禁止继续生成图片
- 当用户视频生成时长超过套餐配额时，触发视频生成功能熔断，禁止继续生成视频
- 熔断后提示用户配额已用尽，引导升级套餐或单独充值算力
- 套餐配额与单独充值算力余额分别计算，互不影响

### 4.12 算力计量与扣费规则
- 用户单独充值的算力余额按火山引擎实际报价扣减
- 每次生成请求完成后，根据实际使用的模型与生成时长或数量计算费用
- 扣费示例：使用 Seaweed 2.0 模型生成 5 秒视频，扣除 4 元
- 系统记录每次生成请求的模型名称、生成时长或数量、扣费金额、生成时间
- 用户实际用量需与火山引擎账单核对，确保计费准确性
- 算力余额不足时限制功能使用并提示充值

### 4.13 充值记录与后台监控规则
- 用户每次单独充值算力时，系统记录付款账单，包含：
  - 用户 OpenID
  - 充值金额
  - 充值时间
  - 支付状态
  - 订单号
- 后台管理功能需展示所有用户充值总额统计
- 后台需实时监控火山引擎账户余额
- 当火山引擎余额低于预设阈值时，触发提醒通知管理员及时充值火山引擎
- 充值记录支持按时间、用户、金额等维度筛选与导出

### 4.14 会员权益规则
- 不同会员等级对应不同的素材包生成次数、趋势研究次数、图片生成额度、视频生成额度、支持平台数量
- 会员到期后自动降级为免费版
- 充值后立即生效

### 4.15 对话记录存储规则
- 所有对话记录需持久化存储
- 对话记录存储在腾讯云 COS 中
- 支持按用户 ID 查询历史记录
- 记录对话时间、内容、消耗 Token 数
- 对话记录在素材资产库中展示

### 4.16 页面导航规则
- 底部导航栏在所有主要页面保持可见
- 当前页面在导航栏中高亮显示
- 支持在 Luna 工作台、平台介绍、定价采购、客服、我的之间快速切换
- 定价与采购界面包含返回按钮

### 4.17 素材导出规则
- 用户在素材资产库中可导出 JSON 格式数据
- 导出操作：向 OpenClaw 发送固定格式化命令
- OpenClaw 返回符合规范的 JSON 数据

### 4.18 COS 存储系统对接规则
- 素材资产库功能对接腾讯云 COS 对象存储系统
- 拉取用户存储在 COS 中的所有文件
- 按文件类型进行前端分类展示
- 对话历史记录同样存储在 COS 中并在素材资产库中展示

### 4.19 COS 用户空间初始化规则
- 用户付费后触发空间初始化流程
- 微信支付回调触发后，Hermes Agent 后端在 COS 写入 users/{openid}/.meta，标记该用户空间已开通
- 数据库记录用户付费状态与开通时间
- 路径结构：
  - users/{openid}/uploads/ 存储用户上传原始文件
  - users/{openid}/outputs/ 存储 AI 处理结果
  - users/{openid}/.meta 存储空间元信息（付费时间、套餐等）

### 4.20 COS 临时凭证获取规则
- 小程序调用 Hermes Agent 接口 POST /api/cos/credential 请求临时凭证
- 后端先验证用户付费状态（查数据库）
- 验证通过后，用 SecretId 与 SecretKey 调用腾讯云 STS 接口
- STS 生成仅限 users/{openid}/* 路径的临时 Token，有效期 30 分钟
- 返回给小程序：tmpSecretId、tmpSecretKey、sessionToken、expiredTime

### 4.21 COS 文件上传规则
- 小程序拿到临时凭证后，直接上传到 COS，不经过 Hermes Agent 服务器
- 上传路径强制为 users/{openid}/uploads/xxx
- 临时凭证过期后自动重新请求，用户无感知
- 其他用户的 Token 无法访问别人的路径（STS 策略硬限制）

### 4.22 COS AI 处理结果写回规则
- Hermes Agent 用主凭证（SecretId 与 SecretKey）处理完后
- 将结果写入 users/{openid}/outputs/xxx
- 小程序用临时凭证读取自己的 outputs/ 目录即可

### 4.23 COS 权限边界规则
- Hermes Agent 后端：使用 SecretId 与 SecretKey（永久），可访问全部路径
- 小程序用户 A：使用 STS 临时 Token（30分钟），仅可访问 users/{openid_A}/*
- 小程序用户 B：使用 STS 临时 Token（30分钟），仅可访问 users/{openid_B}/*
- 未付费用户：无凭证，拒绝访问

### 4.24 后台管理规则
- 后台管理功能集成在平台内部
- 无需单独提供后台管理网址
- 管理功能通过平台统一入口访问

### 4.25 套餐购买与空间开通联动规则
- 用户购买任意付费套餐后，自动触发 COS 空间初始化流程
- 套餐购买成功后，系统自动执行以下操作：
  - 在数据库中记录用户付费状态、套餐类型、购买时间、有效期
  - 在 COS 中创建 users/{openid}/.meta 文件，写入套餐信息与开通时间
  - 初始化用户专属目录结构：users/{openid}/uploads/ 与 users/{openid}/outputs/
- 套餐到期后，用户降级为免费版，但 COS 空间保留，历史数据不删除
- 用户重新购买套餐后，恢复对 COS 空间的完整访问权限

### 4.26 单独充值算力与空间访问规则
- 用户单独充值算力不触发 COS 空间初始化
- 单独充值算力仅用于 AI 生成功能的费用扣减，不影响 COS 访问权限
- 用户必须购买付费套餐后，才能获得 COS 空间访问权限
- 未购买付费套餐的用户，即使充值算力，也无法访问 COS 存储的素材与对话记录

### 4.27 配额消耗与计费分离规则
- 套餐配额消耗：用户使用套餐内的素材包生成次数、趋势研究次数、图片生成额度、视频生成额度时，不扣减算力余额，仅扣减套餐配额
- 算力余额扣减：用户单独充值的算力余额仅在套餐配额用尽后使用，按火山引擎实际报价扣减
- 配额与算力余额独立计算，互不影响
- 系统需明确区分套餐配额消耗与算力余额扣减的触发条件与计算逻辑

### 4.28 火山引擎账单核对与异常处理规则
- 系统每日自动拉取火山引擎账单数据
- 将系统记录的用户实际用量与火山引擎账单进行核对
- 核对维度包括：模型名称、生成时长或数量、扣费金额、生成时间
- 若发现用量与账单不符，系统自动记录异常日志，包含：
  - 用户 OpenID
  - 异常时间
  - 系统记录用量
  - 火山引擎账单用量
  - 差异金额
- 触发人工核对流程，通知管理员处理异常
- 核对结果需记录在数据库中，支持按时间、用户、异常类型筛选与导出

### 4.29 支付回调与订单状态同步规则
- 微信支付成功后，微信服务器向 Hermes Agent 后端发送支付回调通知
- 后端接收回调后，执行以下操作：
  - 验证回调签名，确保请求来自微信服务器
  - 根据订单号查询订单信息，确认订单状态为待支付
  - 更新订单状态为已支付，记录支付时间与支付金额
  - 根据订单类型执行对应操作：
    + 套餐购买：更新用户会员等级、有效期、套餐配额，触发 COS 空间初始化
    + 单独充值：更新用户算力余额，记录充值流水
  - 向微信服务器返回成功响应，确认回调已处理
- 若回调处理失败，系统记录异常日志，触发人工处理流程
- 支持支付回调重试机制，避免因网络异常导致订单状态不同步

### 4.30 退款流程与余额回退规则
- 用户申请退款时，系统执行以下操作：
  - 验证订单状态为已支付，且在退款有效期内
  - 根据订单类型执行对应操作：
    + 套餐购买退款：检查用户是否已使用套餐配额，若已使用则拒绝退款，若未使用则执行退款
    + 单独充值退款：检查用户是否已使用算力余额，若已使用则拒绝退款，若未使用则执行退款
  - 调用微信支付退款接口，发起退款请求
  - 更新订单状态为已退款，记录退款时间与退款金额
  - 回退用户会员等级、套餐配额或算力余额
  - 若为套餐购买退款，删除 COS 空间中的 .meta 文件，标记空间已关闭
- 退款成功后，向用户发送退款成功通知
- 退款失败时，系统记录异常日志，触发人工处理流程

### 4.31 全站合规表达规则
- 统一风控表达：「Luna 基于用户提供素材、公开信息和平台内容规律生成建议。」
- 全站不得出现：自动登录、自动采集后台数据、自动发布、自动推送、Cookie、动态代理、绕过风控、小红书账号绑定、抖音账号绑定

### 4.32 数据库变更规则
- materials 表新增字段：
  - package_config JSONB（存储生成时的配置：mode, platforms, goal, industry）
  - package_result JSONB（存储完整的多平台生成结果）
  - source_mode TEXT（'material' | 'direction'）

## 5. Edge Function luna_guardian 规范

### 5.1 部署路径
supabase/functions/luna_guardian/index.ts

### 5.2 功能说明
1. 接收用户请求，提取用户输入内容与上下文信息
2. 执行安保检查，根据产品边界规则判断请求类型
3. 白名单快速放行：明确命中白名单的请求直接放行，不调用 LLM
4. 黑名单快速拦截：明确命中黑名单 regex 的请求直接拦截，返回安全转向话术
5. 灰色地带语义判断：无法直接判断的请求调用 LLM 进行语义分析，由 LLM 返回处理动作
6. 根据处理动作执行对应操作：
   - allow_chat：正常对话回复
   - allow_generate：生成素材包/内容方案
   - ask_clarify：追问澄清
   - safe_redirect：返回安全转向话术
   - sanitize_output：输出修正
7. 整理用户输入，补全缺失字段
8. 根据请求类型选择 Prompt 模板
9. 调用现有 arkclaw_chat Edge Function（内部 fetch 调用）
10. 检查返回结果是否完整
11. 结果格式不完整时做轻量修复或返回友好提示
12. 结果保存到 materials 表（type=work）

### 5.5 白名单快速放行规则
- 平台白名单：小红书、抖音、视频号、公众号、快手、微博、B站、知乎
- 创作动作白名单：写、生成、改写、润色、优化、起标题、想选题、做脚本、做口播、做封面建议、做标签、做投放建议、做内容规划、做素材包、做内容矩阵、做账号定位
- 内容对象白名单：文案、标题、脚本、选题、口播稿、封面、图片提示词、素材包、内容方案、推广方案、投放建议、标签、话题、栏目、账号定位、人设、卖点、产品介绍、活动信息、品牌资料、用户画像、竞品截图、公开链接、手动上传的数据
- 数据来源白名单：用户手动输入、用户上传图片/视频/文案/表格/截图、用户提供公开链接、公开网页信息、搜索引擎公开结果、平台公开可访问页面、公开榜单/新闻/文章、历史生成结果、素材库里的用户素材
- 命中规则：平台词 + 创作词 → 直接返回 action=allow_generate，不调用 LLM

### 5.6 黑名单快速拦截规则
- 黑名单关键词 regex：登录|扫码|cookie|代理|proxy|采集|爬虫|自动发布|自动推送|账号绑定|selenium|playwright|绕过|风控|后台数据|私有数据|播放量|访客|曝光|转化数据|自动评论|自动私信|批量养号|刷播放|刷粉|刷赞
- 命中规则：用户输入匹配黑名单 regex → 直接返回 action=safe_redirect，不调用 LLM
- 返回话术：「当前 Luna 不支持账号登录、Cookie、代理、后台数据采集、自动发布或自动推送。你可以上传素材，或输入内容方向，我可以基于你提供的素材、公开信息和平台内容规律，帮你生成多平台素材包和投放建议。」

### 5.7 灰色地带语义判断规则
- 触发条件：用户输入既未命中白名单，也未命中黑名单 regex
- 处理方式：调用 LLM 进行语义分析，判断用户意图
- LLM 判断依据：
  - 平台词 + 数据分析词 + 用户提供数据 → allow_generate
  - 平台词 + 数据分析词 + 要 Luna 去抓后台 → safe_redirect
  - 意图不明确 → ask_clarify
- LLM 返回处理动作：allow_chat、allow_generate、ask_clarify、safe_redirect、sanitize_output
